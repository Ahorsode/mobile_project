
Tier 1: Beginner (Foundations)

Lesson 1.1: The Digital Brain
The Theory: A computer’s CPU is often called its "brain," but it only understands the simplest instructions. All data—whether a photo or a line of code—is stored as binary, a series of 0s and 1s representing "off" and "on" electrical states.
+2
The Logic: To run a program, the CPU follows the Fetch-Decode-Execute cycle: it fetches an instruction from memory, decodes it into a command it understands, and then executes that command. This process repeats millions of times per second to make your apps work.
+3
Code Discovery:
Python
# Converting the letter 'A' to its binary (digital) form
char = 'A'
print(bin(ord(char)))

The Logic: In Python, a single misplaced space can cause an "Indentation Error." This forces developers to write well-organized code from the start.

Lesson 2.1: The Variable Vault

The Logic: To create a variable, you use an assignment statement (e.g., x = 10). You can always check a variable's type using the type() function.

Lesson 2.3: The Math Lab

+2

Lesson 4.1: The Infinite Loop
The Theory: A while loop repeats a block of code as long as a condition remains True. It is "condition-controlled" because the loop only stops when the condition becomes False.
+1
The Logic: Be careful! If the condition never becomes False, you create an Infinite Loop, which can crash your program.
+1
Code Discovery:
Python
countdown = 5
while countdown > 0:
    print(countdown)
    countdown -= 1  # Important: change the condition!
print("Blast off!")

Lesson 5.1: The Modular Architect
The Theory: As programs grow, they become harder to manage. Professional developers use functions to break complex code into smaller, manageable "modules". A function is a named block of code that performs a specific task and can be executed whenever needed.
The Logic: Functions allow for code reuse; instead of writing the same logic ten times, you write it once in a function and "call" it ten times. This also makes debugging easier because you only have one place to fix a mistake.

Python

The Theory: Functions can be made more powerful by sending data into them. An argument is a piece of data sent into a function, and a parameter is the special variable that receives it.

Lesson 5.3: The Scope Shield
The Theory: Not all variables are visible everywhere. A local variable is created inside a function and cannot be accessed by code outside that function. Its "scope" is limited to its own function "shield."
The Logic: Global variables are visible to every function in a file. However, relying on them is risky because any part of the program can change them, leading to bugs that are hard to track.

Lesson 7.2: List Methods
The Theory: Python provides built-in methods to manipulate lists dynamically. You can add items with .append(), insert them at a specific spot with .insert(), and remove them with .remove().

Python

levels.remove(2)   # Remove the number 2
print(levels)

Lesson 9.1: The Key-Value Map
The Theory: A dictionary is an object that stores data in key-value pairs. Instead of using a numeric index, you use a unique "key" (like a label) to find its "value".
The Logic: Dictionaries are highly efficient for looking up data. You can retrieve a value using its key inside brackets [] or by using the .get() method, which prevents errors if the key is missing.
Code Discovery:
Python

print(stats.get("Exp", 0)) # Output: 0 (default if not found)

Lesson 9.2: Set Theory
The Theory: A set is a collection of unique elements. If you try to add a duplicate item to a set, Python will simply ignore it.
The Logic: Sets are perfect for mathematical operations like Union (combining two sets) or Intersection (finding only the items that appear in both sets).
Code Discovery:
Python
set1 = {1, 2, 3}
set2 = {3, 4, 5}

print(set1.intersection(set2)) # {3}

The Logic: The self parameter is a reference to the current instance of the class and is used to access variables that belong to the class.

Lesson 10.3: Encapsulation
The Theory: Encapsulation is the practice of hiding the internal state of an object and requiring all interaction to be performed through an object's methods.
+1
The Logic: In Python, you can hide an attribute by starting its name with two underscores (e.g., __hidden_speed). This prevents code outside the class from accidentally changing sensitive data.
+1
Code Discovery:
Python
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance # Private attribute

+1
The Logic: This creates an "is-a" relationship. For example, a Car is-a Vehicle. The subclass can then add its own unique features or "override" existing ones.

Python

Lesson 12.1: The Echo (Recursion)
The Theory: Recursion occurs when a function calls itself to solve a smaller version of the same problem.
+1
The Logic: Every recursive function must have a Base Case (the condition that stops the calling) to prevent an infinite loop of calls that would crash the app.
+1
Code Discovery:
Python
def count_down(n):
    if n == 0: # Base Case
        print("Done!")
    else:
        print(n)
        count_down(n - 1) # Recursive Call
